# Direct File Upload Design

**Date:** 2026-05-14
**Status:** Approved

---

## Goal

Replace the external URL download field with direct file uploads to Supabase Storage. Paid asset downloads are gated behind purchase verification — only buyers receive a signed URL. Free assets generate a signed URL for anyone. Max file size: 50 MB.

---

## Current State

- `digital_products.asset_url` stores a plain text external URL (Google Drive, MEGA, etc.)
- `asset_url` is publicly readable — anyone who can SELECT a published product sees the raw URL
- `ProductDetail.tsx` does `window.open(asset_url)` — no server-side gate
- `Library.tsx` also exposes the raw URL in the Download button
- The `product-images` bucket exists for cover images (no migration — created manually)
- No `product-files` bucket exists yet

---

## Design

### Storage

New private Supabase Storage bucket: **`product-files`**

- Not publicly accessible — no one can fetch a file by URL without a signed token
- File path convention: `{creator_id}/{product_id}/{original_filename}`
- Max upload size enforced client-side: 50 MB

### Database

New column on `digital_products`:

```sql
ALTER TABLE digital_products ADD COLUMN IF NOT EXISTS asset_file_path TEXT;
```

- `asset_file_path` — Supabase Storage path (e.g. `abc123/xyz789/my-shader-pack.zip`)
- `asset_url` — kept as-is for backward compatibility with existing external URL assets
- When both are set, `asset_file_path` takes priority
- When neither is set, publishing is still blocked (existing validation)

### Upload UI (`DashboardAssetDetail.tsx`)

Replace the plain "Download URL" text input with:

```
[ Upload file ]  my-shader-pack.zip (12.4 MB)  [Remove]

— or —

External URL: [___________________________]
```

- Primary: file upload button → uploads to `product-files/{creator_id}/{product_id}/{filename}`
- Secondary: external URL input remains as fallback (shown below upload, collapsed label "or use an external URL")
- If a file is already uploaded, show filename + size + "Replace" button
- If a file is uploaded, `asset_url` is cleared to null on save
- If the uploaded file is removed (clicking "Remove"), `asset_file_path` is cleared to null and the external URL input becomes active again
- Only one source is saved at a time: file upload OR external URL, never both
- Upload happens on form save (not immediately on file select), same pattern as cover image

### Edge Function: `get-download-url`

New Edge Function at `supabase/functions/get-download-url/index.ts`

**Request:**
```ts
POST /functions/v1/get-download-url
Authorization: Bearer <user JWT>
Body: { product_id: string }
```

**Logic:**
1. Fetch `digital_products` row by `product_id` — get `price`, `asset_file_path`, `asset_url`, `status`
2. If `status !== 'published'` → 404
3. If `asset_file_path` is null → return `{ url: asset_url }` (external URL fallback)
4. If `price === 0` (free) → generate signed URL, return it
5. If `price > 0` (paid) → query `purchases` for `buyer_id = auth.uid()` AND `product_id` AND `status = 'completed'`
   - Found → generate signed URL, return it
   - Not found → 403 `{ error: 'Purchase required' }`
6. Signed URL TTL: 3600 seconds (1 hour)

**Response:**
```ts
{ url: string }  // signed URL or external URL
```

Uses service role key to call `supabase.storage.from('product-files').createSignedUrl(...)`.

### Download flow (`ProductDetail.tsx` + `Library.tsx`)

Replace `window.open(asset_url, '_blank')` with:

```ts
const res = await fetch('/functions/v1/get-download-url', {
  method: 'POST',
  headers: { Authorization: `Bearer ${session.access_token}`, 'Content-Type': 'application/json' },
  body: JSON.stringify({ product_id: product.id }),
});
if (!res.ok) { toast.error('Download unavailable'); return; }
const { url } = await res.json();
window.open(url, '_blank', 'noopener,noreferrer');
```

Show a loading spinner on the Download button while the request is in flight.

### Storage RLS / Bucket Policy

`product-files` bucket:
- **Public access: OFF** — no anonymous reads
- **No direct RLS policies needed** — files are only accessed via signed URLs generated by the Edge Function (service role bypasses RLS)

### `useMarketplace.ts` types

Add `asset_file_path?: string` to `useCreateProduct` and `useUpdateProduct` mutation types.

## Out of Scope

- Chunked / resumable uploads (files > 50 MB) — not needed for this scope
- Multiple files per asset (e.g. separate Windows/Mac/Linux builds) — separate feature
- Download count tracking — separate feature
- File versioning — separate feature
- Virus scanning — separate feature

---

## Security Notes

- `asset_url` (external URLs) remains publicly readable in the DB — this is acceptable for external links since the creator chose to use an external host
- `asset_file_path` is a storage path, not a URL — useless without a signed token
- Signed URLs expire after 1 hour — buyers must re-request if they need to download again after expiry
- The Edge Function uses `supabase.auth.getUser()` to verify the JWT — unauthenticated requests are rejected
- Free assets: no auth check on purchase, but auth is still required (must be logged in) to call the function
